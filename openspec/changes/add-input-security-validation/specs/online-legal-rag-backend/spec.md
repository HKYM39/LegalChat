## ADDED Requirements

### Requirement: 在线后端必须在问答入口执行权威输入安全校验
系统 MUST 在 `POST /ask` 接口完成请求解析后、进入检索与模型调用前执行权威输入安全校验。任何客户端即使绕过前端，本接口也 MUST 主动拒绝命中安全规则的请求。

#### Scenario: 后端拦截绕过前端的非法输入
- **WHEN** 客户端直接调用 `POST /ask` 并提交命中输入安全规则的 payload
- **THEN** 后端在进入检索和模型调用前拒绝该请求

#### Scenario: 后端放行符合规则的问答请求
- **WHEN** 客户端向 `POST /ask` 提交符合输入安全边界的正常法律问题
- **THEN** 后端允许请求继续进入既有问答链路

### Requirement: 在线后端必须返回可识别的输入安全错误
系统 MUST 为输入安全拒绝返回可识别的结构化错误响应，至少包含固定错误类型、原因码与面向前端展示的提示信息，使客户端能够稳定区分安全拒绝与其他接口错误。

#### Scenario: 返回结构化安全错误
- **WHEN** 后端将请求判定为输入安全拒绝
- **THEN** `POST /ask` 响应中包含明确的输入安全错误类型与原因码

#### Scenario: 普通错误不伪装为安全错误
- **WHEN** `/ask` 因非输入安全原因失败
- **THEN** 系统返回既有错误语义，而不是错误地标记为输入安全拒绝
