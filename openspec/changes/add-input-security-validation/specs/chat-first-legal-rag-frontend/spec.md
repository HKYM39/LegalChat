## ADDED Requirements

### Requirement: 前端必须在聊天提交前执行输入安全预检
系统 MUST 在聊天 composer 的提交路径中执行输入安全预检。当前端判定输入命中安全拒绝规则时，MUST 阻止本次提交，且不得向 `POST /ask` 发送请求。

#### Scenario: 本地预检拦截非法输入
- **WHEN** 用户在聊天输入框提交包含明显攻击型 payload 的内容
- **THEN** 前端立即阻止提交，并保持请求未发送到后端

#### Scenario: 本地预检放行正常问题
- **WHEN** 用户在聊天输入框提交符合安全边界的正常法律问题
- **THEN** 前端允许请求继续进入既有提交流程

### Requirement: 前端必须明确区分安全拒绝与普通请求失败
系统 MUST 将输入安全拒绝呈现为独立于网络失败、限流或通用异常的错误状态。无论是本地预检命中，还是后端返回安全拒绝错误，界面都 MUST 展示清晰一致的提示。

#### Scenario: 显示本地安全拒绝提示
- **WHEN** 前端本地预检拒绝当前输入
- **THEN** composer 展示明确的输入安全提示，而不是通用“请求失败”消息

#### Scenario: 显示后端安全拒绝提示
- **WHEN** 前端提交后收到后端返回的输入安全错误
- **THEN** 系统将该错误映射为与本地预检一致的安全提示样式与语义
