## ADDED Requirements

### Requirement: 输入安全校验必须定义统一的聊天输入边界
系统 MUST 为聊天输入提供统一的安全校验边界，并在前端与后端使用一致的输入规范化与判定语义。该边界至少覆盖空白输入、超长输入、控制字符或二进制片段，以及明显网络攻击型 payload。

#### Scenario: 判定正常法律问题可通过
- **WHEN** 用户提交普通自然语言法律研究问题，且内容不包含异常控制字符、超长片段或明显攻击载荷
- **THEN** 输入安全校验结果标记为允许通过，不触发安全拒绝

#### Scenario: 判定明显攻击型 payload 被拒绝
- **WHEN** 用户提交包含脚本注入、协议探测、命令注入、SQL 注入、路径穿越或等价网络攻击特征的 payload
- **THEN** 输入安全校验结果标记为拒绝，并返回统一原因码与用户可读提示

### Requirement: 输入安全校验必须返回统一的结构化拒绝语义
系统 MUST 为被拒绝的输入返回统一的结构化语义，至少包含固定错误类型、原因码与面向用户的提示信息，确保前端本地拦截与后端接口拒绝可以共享同一套表达。

#### Scenario: 前端生成本地安全拒绝状态
- **WHEN** 前端在提交前命中输入安全规则
- **THEN** 前端获得可直接展示的结构化拒绝结果，并据此阻止发送请求

#### Scenario: 后端生成服务端安全拒绝响应
- **WHEN** 后端在接口入口命中输入安全规则
- **THEN** 后端返回与前端同语义的结构化错误字段，而不是通用 500 或不可识别错误

### Requirement: 输入安全拒绝不得进入问答下游处理链路
系统 MUST 在输入安全校验失败时终止后续问答处理。被拒绝请求不得进入检索、向量查询、模型调用与正常问答日志主链路。

#### Scenario: 安全拒绝发生在问答前
- **WHEN** 输入安全校验将请求判定为拒绝
- **THEN** 系统在任何检索、重排或模型调用发生前终止该请求

#### Scenario: 安全拒绝不写入正常问答日志
- **WHEN** 请求因为输入安全规则被拒绝
- **THEN** 系统不会把该请求作为成功或失败问答写入正常问答日志记录
