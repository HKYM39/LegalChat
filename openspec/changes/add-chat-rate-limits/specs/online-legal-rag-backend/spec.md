## ADDED Requirements

### Requirement: 在线后端必须在问答入口执行可配置聊天限流
系统 MUST 在 `POST /ask` 请求进入检索、向量查询或模型生成前执行聊天频率限制校验，并使用环境变量控制分钟级与天级阈值。任一窗口超限时，系统 MUST 立即终止问答处理。

#### Scenario: 超限请求在问答前被拦截
- **WHEN** 客户端向 `POST /ask` 提交请求，且当前聊天主体已经达到分钟级或天级上限
- **THEN** 系统在调用检索层和模型层之前返回限流错误，而不是继续执行问答链路

#### Scenario: 未超限请求继续进入问答链路
- **WHEN** 客户端向 `POST /ask` 提交请求，且当前聊天主体仍在分钟级与天级额度内
- **THEN** 系统允许该请求继续执行后续检索与回答生成流程

### Requirement: 在线后端必须暴露可识别的聊天限流错误
系统 MUST 为聊天超限返回可识别的结构化错误响应，至少区分分钟级限制和天级限制，并为前端提供可直接消费的提示字段或等价信息。

#### Scenario: 返回分钟级限流错误
- **WHEN** 客户端命中分钟级聊天上限
- **THEN** `POST /ask` 响应明确标识该错误属于聊天限流，且窗口类型为分钟级

#### Scenario: 返回天级限流错误
- **WHEN** 客户端命中天级聊天上限
- **THEN** `POST /ask` 响应明确标识该错误属于聊天限流，且窗口类型为天级
